Criação do protótipo 

1. Estrutura Geral dos Dados
Você terá um arquivo JSON (data.json) que conterá todas as informações sobre voluntários, organizadores, e projetos. 

A estrutura básica dos dados será organizada em três seções principais:
Voluntários: Contém a lista de voluntários cadastrados.
Organizadores (ONGs): Contém a lista de ONGs ou organizadores que criam projetos.
Projetos: Contém a lista de projetos que podem ser criados pelos organizadores e onde os voluntários podem se inscrever.

2. Atributos para Cada Entidade
Cada entidade (Voluntário, Organizador, Projeto) terá atributos específicos.

Voluntário
id: Identificador único para cada voluntário.
username: Nome de usuário do voluntário.
email: Email do voluntário.
keywords: Lista de palavras-chave que representam os interesses do voluntário (ex.: ["educação", "meio ambiente"]).

Organizador (ONG)
id: Identificador único para cada organizador.
name: Nome da ONG ou organizador.
email: Email da ONG.
projects: Lista de projetos criados por essa ONG (será uma lista de dicionários com os projetos).

Projeto
id: Identificador único para cada projeto.
title: Título do projeto.
description: Descrição do projeto.
organizer_id: ID do organizador que criou o projeto (referência cruzada com a ONG).
keywords: Lista de palavras-chave que descrevem o projeto (ex.: ["meio ambiente", "sustentabilidade"]).


3. Funcionalidades e Operações
Para implementar o protótipo, terá as seguintes operações básicas:

1. Cadastro de Voluntário
Voluntários podem se cadastrar fornecendo um nome de usuário, email, e palavras-chave de interesse.
A função deve gerar um id único para o voluntário e salvá-lo no JSON.

2. Cadastro de Organizador (ONG)
Organizações podem se cadastrar fornecendo o nome, email, e criar uma lista de projetos.
A função deve gerar um id único para a ONG e salvá-la no JSON.

3. Criação de Projetos
Organizações podem criar projetos, fornecendo título, descrição, palavras-chave, e o id do organizador.
A função adiciona o projeto à lista de projetos da ONG e o armazena no JSON.

4. Busca de Projetos
Voluntários podem buscar projetos com base em palavras-chave.
A função deve retornar uma lista de projetos que contenham as palavras-chave de interesse do voluntário.
